# SCORING ─ 得点計算システム設計

> パワフル漫才の採点システムの根幹。素の得点化(威力×リズム×能力)＋コンボ修正までを定義する。
> 上流(ネタ構成=`03_NETA.md`)→ 本書(得点化・コンボ修正)→ 今後(審査員システム=決勝)の順で積む三位一体設計。
> 関連: ネタ構成=`03_NETA.md` / 漫才プレイ(リズムゲーム)=`02_MANZAI_PLAY.md` / 賞レース=`01_SUCCESS.md`

---

## 0. 全体構造（コンボなし）

```
パーツ実得点  s_i = 威力_i × リズム判定倍率 × 基礎能力倍率
ネタ実得点     S  = Σ s_i      （全スロットのパーツ実得点の合計）
手応え         T  = Σ 威力_i    （プレイヤーに見える概算期待値）
```

- **手応え T**：プレイヤーがネタ編成画面で見る数値。「概ねこれくらいの点が平均的に期待できる」という目安。**威力の単純合計**。
- **ネタ実得点 S**：実際の漫才プレイ後に算出される最終得点。リズムの出来と主人公の能力で T から増減する。
- 設計の肝：**平均的なプレイ × 平均的な能力(50)のとき、S ≒ T になる**。これによりプレイヤーは手応えを信頼して編成できる。

---

## 1. 威力（基礎点）✅

- 各パーツは**威力【1〜9】**を持つ（`03_NETA.md`)。これがそのパーツの基礎点。
- 9スロット全部の威力合計が「**手応え T**」＝プレイヤー表示の概算値。
- 威力はネタ編成（パーツ選び）で決まる固定値。プレイや能力では変動しない（変動するのは下記の倍率）。

---

## 2. リズム判定倍率 ✅

漫才プレイ（リズムゲーム）の各パーツのタイミング判定で、そのパーツの得点に倍率がかかる。

| 判定 | 倍率 |
|------|------|
| Perfect | **1.30** |
| Good | **1.00** |
| Safe | **0.70** |
| Bad | **0.30** |

- **設計根拠**：「平均的なプレイ」の判定分布(下記)での**期待倍率が約1.0**になるよう設定。これが「手応え≒平均実得点」を成立させる。
- 平均的プレイの判定分布（設計上の基準値）：Perfect 30% / Good 45% / Safe 20% / Bad 5%
  - 期待倍率 = 1.30×0.30 + 1.00×0.45 + 0.70×0.20 + 0.30×0.05 = **0.995 ≒ 1.0** ✅
- Good=1.00 を基準点に、Perfectで上振れ(+30%)、Safe/Badで下振れ。プレイの巧拙が得点に直結する。

---

## 3. 基礎能力倍率 ✅

主人公の基礎能力（ボケ/ツッコミ/大喜利/トーク/演技、各0-100）に応じて、パーツ得点が底上げされる。

### 3.1 能力スコア → 倍率の変換
```
基礎能力倍率 = 0.80 + (能力スコア / 100) × 0.40
```
| 能力スコア | 倍率 |
|-----------|------|
| 0 | 0.80 |
| 25 | 0.90 |
| **50** | **1.00（基準）** |
| 75 | 1.10 |
| 100 | 1.20 |

- **能力50で倍率1.0**＝平均的能力なら手応え通り。育成で±20%の幅。
- やり込み（能力育成）は効くが、リズムの出来を完全に覆すほどではない範囲に抑え、両輪を維持。

### 3.2 パーツのタグ別 能力参照配分 ✅
パーツの**性質タグごとに、参照する能力とその比率（合計10）**が決まる。これにより「能力スコア(0-100)」を算出。
```
能力スコア = Σ( 参照能力値 × 比率/10 )
```

#### 能力体系（`03_NETA.md`と対応）
- **共通能力**（両形式で参照）：ボケ / ツッコミ / 大喜利
- **形式固有枠 "f"**：しゃべくり版なら**トーク**、コント版なら**演技**に割り当てる
- **形式共有パーツ**（ツカミ・オチ）：形式固有を使う場合は「トークと演技の平均(t/a平均)」を参照

#### 配分テーブル（合計10）

**ツカミ（形式共有・2種）**
| タグ | 能力配分 |
|------|----------|
| 自己紹介 | t/a平均7 : ボケ3 |
| ギャグ | ボケ7 : 大喜利3 |

**フリ（形式分離・4種）** ※"f"=しゃべくりはトーク/コントは演技
| タグ | 能力配分 |
|------|----------|
| 日常 | f5 : ボケ3 : 大喜利2 |
| 説明 | 大喜利4 : f4 : ボケ2 |
| 持論 | 大喜利5 : f4 : ツッコミ1 |
| 妄想 | 大喜利7 : f2 : ボケ1 |

> ※「時事」フリは削除(特能◯時事として扱う)。「妄想」フリを新設(大喜利7=発想力の主役)。
> 説明・持論は大喜利偏重を緩和(大喜利6→4/5、fを3→4)し、妄想が大喜利の頂点になるよう序列化。

**ボケ（形式分離・5種）**
| タグ | 能力配分 |
|------|----------|
| あるある | ボケ5 : f3 : 大喜利2 |
| 誇張 | ボケ7 : f3 |
| 裏切り | ボケ6 : 大喜利2 : f2 |
| 動き | ボケ4 : f6（コント版=演技で特に活きる） |
| ひねり | ボケ4 : 大喜利4 : f2 |

**ツッコミ（形式分離・5種）**
| タグ | 能力配分 |
|------|----------|
| 直球 | ツッコミ7 : f3 |
| 例え | ツッコミ5 : 大喜利4 : f1 |
| ノリ | ツッコミ4 : f4 : ボケ2 |
| 説明 | ツッコミ5 : 大喜利4 : f1 |
| スカシ | ツッコミ5 : 大喜利2 : f3 |

**オチ（形式共有・2種）**
| タグ | 能力配分 |
|------|----------|
| 回収 | 大喜利6 : ボケ4 |
| 裏切り | ボケ6 : 大喜利2 : t/a平均2 |

> **設計の確認（シミュレーション済み）**：
> - 全17タグとも比率合計10、能力オール50で倍率1.0（基準）を確認 ✅
> - 形式固有枠の大きいタグ(動き/ノリ/日常)ほど、しゃべくり版とコント版で倍率差が顕著。
>   例: しゃべくり特化(トーク90)なら日常フリ・動きボケのしゃべくり版が1.128、コント版は1.008。
>   コント特化(演技90)なら動きボケ・コント版が1.128で最大化。→ ビルドと形式・タグ選択の戦略性が成立。
> - 「動き×コント×演技」が噛み合う設計＝演技力が死にパラメータにならない。

## 5. 賞レース突破判定（準決勝まで）✅

準決勝までは**絶対得点・閾値方式**。ネタ実得点 S が各ラウンドの閾値以上なら突破。

| ラウンド | 突破閾値（仮値） |
|---------|-----------------|
| 1回戦 | 40点以上 |
| 準々決勝 | 48点以上 |
| 準決勝 | 55点以上 |

- 閾値は上記サンプルネタ（手応え47）を基準にした**仮値**。実際は「年次・その時期のネタの手応え水準・成長曲線」に合わせて調整する。
- 意味づけ：育成不足＋プレイ下手では1回戦突破も厳しい。上位ラウンドは「良いネタ(高い手応え)＋良い能力＋良いプレイ」の総合力が要る。
- **決勝のみ別系統**＝相対評価＋審査員システム（本書の範囲外。次層で設計）。

## 6. コンボによる得点修正 ✅（設計完了）

コンボの詳細(ラインナップ・発動条件)は `03_NETA.md` §3.5。得点計算上の扱いは以下。

- **コンボ倍率は威力(基礎点)に乗算**する。リズム判定・能力倍率をかける**前**の段階で効く。
  ```
  パーツ実得点 = (威力 × コンボ倍率) × リズム判定倍率 × 基礎能力倍率
  ```
  これにより、コンボ込みの「手応え」を編成画面に表示でき、かつコンボもリズムで決めないと活きない。
- **倍率値**: 小コンボ ×1.10 / 中コンボ ×1.25 / 役満 ×1.15。
- **小・中コンボ(局所・厚い)**: 該当パーツのみに適用。各パーツは関与コンボの**最上位のみ**(小と中は乗算しない)。
- **役満(全体・薄い)**: 成立時、**全9パーツ**に×1.15。さらに小・中と**両立し乗算**(例: 役満×中=1.15×1.25=1.4375)。
- **Bad消滅**: コンボ関与パーツがBad判定だと、そのコンボは不成立(倍率なし)。Bad以外は満額。
  - 倍率満額でも、リズム判定倍率は別途各パーツにかかる(Perfectで決めるほど最終得点は伸びる)。
- **計算順序**: ①配置からコンボ候補を判定 → ②リズムプレイ実行(各パーツPerfect〜Bad) → ③コンボ成立確定(関与パーツにBadあれば不成立) → ④成立コンボの倍率を威力に適用 → ⑤リズム判定倍率・能力倍率を乗算 → ⑥合計。
- **表示と計算の分離**: 成立全コンボを表示(内包小コンボ含む)するが、倍率計算は上記ルール。

### 最終得点の上限 = 100点 ✅
M-1グランプリがモデルのため**100点が満点**。倍率計算の結果がどうなっても、最終的に表示・判定に使う得点は100点を超えない(暴走防止の安全装置)。
※100点満点スケールへの詳細な換算(威力のスケーリング、準決勝までの閾値の100点換算)は、下記の数値バランス調整で詰める。
