# パワフルマンザイ ドット絵パイプライン仕様（決定論・生成AI不要）

目的: キャラの**表情・ポーズ変更を「絵柄を変えず」コードだけで量産**する。
生成AI（fal.ai/Gemini）に毎回頼らない。手順を固定し、安いモデル/スクリプトでも再現可能にしてトークンを節約する。

## 絶対ルール（厳守）
- キャンバスは **64×96 固定**。テキストマップの**全行が正確に64文字**であること（レンダラが検証）。
- 使う記号は**そのキャラのパレット記号のみ**（`tools/pxpalettes.ps1`）。
- **鼻を描かない・口を描かない**。顔は**目＋眉のみ**。目と目の間〜あご先に、肌以外の点があれば肌（`S`）か肌影（`1`）で消す。
- スタイル: **フラット＋1段影**、**1px の馴染む濃色輪郭（記号`K`、純黒ではない）**。

## パレット記号（例: hero）
`.`=透過 / `K`=輪郭(暖かい濃色) / `H`=髪 / `2`=髪ハイライト / `S`=肌 / `1`=肌影 /
`W`=白(シャツ/白目) / `9`=シャツ影 / `R`=赤スーツ / `3`=赤影 / `T`=ネクタイ黄 / `C`=頬紅 / `5`=靴赤茶
（他キャラは `tools/pxpalettes.ps1` を参照。yada=青B/4, person1=紫P/5+眼鏡G, person2=茶N/6+ひげD/7）

## フェーズ
- **Phase 0** パレット確定（`tools/pxpalettes.ps1`）
- **Phase 1** 下絵の自動生成（`tools/underdraw.ps1`）… 元マスターを縮小・減色・α2値化・1px輪郭
- **Phase 2** 手clean（下絵をなぞって整える）… 本仕様書の主対象
- **Phase 3** 確定（`_pxmap_<char>.txt` をマスターとして確定）
- **Phase 4** 表情差分（目＋眉のブロックのみ差し替え）
- **Phase 5** パーツ辞書化（頭/目/眉/髪/胴/腕/手/靴/眼鏡/ひげ を辞書化 → 下絵なしで組立描画）

## ツールと使い方（Windows PowerShell 5.1・ASCIIのみの.ps1）
- 下絵生成:
  `powershell -ExecutionPolicy Bypass -File tools\underdraw.ps1 -char <c> -master <masterPng>`
  → `_pxmap_<c>.txt` を生成
- 描画（レンダリング＋検証＋8倍プレビュー）:
  `powershell -ExecutionPolicy Bypass -File tools\pxrender.ps1 -map _pxmap_<c>.txt -pal <c> -scale 8`
  → `_pxmap_<c>.png`（実寸64×96） と `_pxmap_<c>._big.png`（8倍・目視確認用）
  → エラーなく完了＝「全行64文字・既知記号のみ」を保証

## Phase 2 手clean 手順（この順で）
1. まず現状をレンダリングし `_pxmap_<c>._big.png` を Read してベースラインを把握。参照は元絵 `images/parts/<c>_master.png`。
2. **ノイズ除去**: 透過領域に浮いた孤立点（周囲が`.`だけの単独の`H`/`K`等）を`.`に。髪トップの飛び散りを整理。
   - 機械的作業はPowerShellの小スクリプトで一括処理してよい（例: 8近傍が全て`.`の点を除去）。
3. **輪郭整形**: シルエットの外周を1pxの`K`で連続させる。途切れを補修、二重線を1pxに。
4. **目・眉**: 各目＝小さな暗ブロック＋`W`のハイライト1px。眉＝目の上の短い`K`。左右対称に。
5. **鼻・口ゼロ保証**: 目の下〜あごの範囲を点検し、肌以外（あご輪郭`K`を除く）は`S`/`1`で消す。
6. **1段影**: 上着の左/下エッジに`3`、あご下・髪の生え際下の肌に`1`を軽く足す。
7. 変更ごとに再レンダリングして `_big.png` を Read し確認。崩れたら戻す。

## 品質基準（QC・完了条件）
- `pxrender.ps1` がエラーなく完走（64×96・既知記号）。
- **鼻・口が無い**。顔は目＋眉のみ。
- シルエット輪郭が1pxで連続、浮遊ノイズが無い。
- 主人公と識別可能で、パレットが一貫（色ブレ無し）。
- 元絵の特徴（髪型・スーツ色・ネクタイ・手・靴）が保持されている。

## 成果物
- `_pxmap_<char>.txt`（編集可能キャンバス＝確定マスター）
- `_pxmap_<char>.png` / `_pxmap_<char>._big.png`
- 変更内容の短い報告

## やってはいけないこと
- キャンバス寸法を変える / 未定義記号を使う / 鼻・口を描く。
- 他のファイルを書き換える・git commit する。
- 生成AI（fal.ai等）を呼ぶ（このパイプラインは非依存）。
