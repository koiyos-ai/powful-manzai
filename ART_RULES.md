# パワフル漫才 キャラ作画ルール ＆ QCチェック項目（固定）

fal.ai（Flux Kontext で編集 → BiRefNet で透過）で生成するキャラ差分は、**毎枚この基準で精査してから採用・報告する**。目視だけでは見落とすため、チェック項目を必ず1つずつ確認する。

## 1. パワプロ風 作画ルール（厳守・不変）
1. **2頭身**（頭：体 ≈ 1：1）。基準キャラの頭身を一切変えない。
2. 顔に **鼻・口・耳を描かない**。表情は **目・眉・頬の赤み**だけで表現する。
3. 手は **指のない丸いミトン状**。**拳・指・爪・関節線を描かない**（手を上げても丸いまま）。
4. **フラットなアニメ塗り**：太い濃紺の輪郭線＋シンプルなセル影。髪・靴の光沢も基準と同じ塗り。
5. **配色固定**：主人公＝赤スーツ／黄ネクタイ、矢田＝青スーツ／黄ネクタイ。肌・髪・靴・シャツは基準色に一致させる。
6. 全身・正面・中央配置・**純白 or 透過**背景。キャラの表示サイズも基準と同じ。
7. **1体だけ**。クローン・小物・文字・サングラス等を足さない。

## 2. 顔とポーズの分離生成 ＆ 組み合わせ（紙芝居方式・パワプロ準拠）
サクセスイベントは紙芝居式。**表情（顔）とポーズ（体）を別々に生成し、実行時に重ねて組み合わせる**。組み合わせ前提なので、表情とポーズの全組み合わせを1枚ずつ描かない。

- **構成**：ポーズ N種（体＝頭の位置・大きさ固定）＋ 表情 M種（顔プレート＝目・眉・頬のみ）→ **N×M通り**。例：ポーズ5＋表情5＝**10枚で25パターン**。
- **必須条件**：
  1. 全ポーズで **頭の位置・大きさを固定**（生成後に基準位置へ自動整列する）。
  2. **顔プレートは基準の頭位置に登録**し、どのポーズにも同じ座標で重なるようにする。
  3. 顔は常に **基準プレート**を使う＝顔の色・形がドリフトしない（後述の顔ロックと同義）。
- **実装**：success-play 等は「**体img＋顔img を CSS で重ねる**」だけ。リズムゲームには非干渉。
- **命名規則**：体＝`body_<キャラ>_<pose>.png`、顔＝`face_<キャラ>_<expr>.png`、基準ベース＝`<キャラ>_canon.png`。

## 3. QCチェック項目（生成ごとに全項目 PASS で初めて採用）
| # | 項目 | 確認方法 |
|---|---|---|
| A | 口・鼻・耳がない | 顔を拡大して目視 |
| B | 手が指なしの丸いミトン（拳/指/爪/関節線なし） | **両手を拡大して目視** |
| C | 頭身が基準と一致 | bboxアスペクト比＋全身目視（自動頭身計測はポーズ差で破綻するので使わない） |
| D | 肌・髪・スーツ・ネクタイ・靴の色が基準と一致 | 色正規化（基準パレットへスナップ）＋目視 |
| E | 髪・靴の光沢・塗りが基準と一致 | 拡大目視 |
| F | 同一人物（顔・髪型が基準と一致） | 基準と並べて目視 |
| G | 透過がクリーン（連結成分=1、縁ノイズなし） | 連結成分カウント＋濃色背景に合成して目視 |
| H | 1体だけ・余計な要素なし | 全体目視 |

## 4. Kontext のドリフト対策（恒久）
Kontext は画像全体を再生成するため、毎回わずかに ①手の指 ②色・光沢 ③頭身 が変動する。これを潰すための工程：
1. **顔ロック**：紙芝居方式なので **顔は常に基準の顔プレートを重ねる**＝顔の色・形は絶対にドリフトしない。
2. **色ロック**：生成後に **基準パレットへ色スナップ（正規化）** し、肌／スーツ／髪／靴を完全一致させる。
3. **手**：プロンプトで「丸いミトン・指/拳/関節の線を描かない」を最重要化。Bゲートで毎回確認し、NGは再生成。安定しなければ **基準のミトン手を合成**して上書き。
4. **頭身ロック**：体は頭位置基準で整列。明らかな頭身ズレは再生成。
5. **QCゲート**：A〜H を全て通過した枚だけ採用し、結果を表で報告する。

## 5. 制作チェックは「5人評議会」で行う（取りこぼし防止・最重要）
チェックリスト1本では視野狭窄で必ず取りこぼす（例：ポーズばかり直して「**表情が全部同じ**」を見落とした）。すべての成果物は、**視点の異なる5人が各自の担当チェックリストで独立に審査**する。**1人でも不合格なら採用しない。** 報告は必ず5人の評定表で出す。各評議員は越権せず**自分の担当だけを厳しく**見る（責任の所在を明確化＝取りこぼし防止）。

### 評議員1 ゲームディレクター ─ 目的・大局・スコープ
- [ ] 依頼の**本来の目的**に直結しているか（目的が「表情豊かなキャラ」なら、表情が無い時点で不合格）
- [ ] **必要要素が数で揃っているか**（表情の種類数・ポーズ数・対象キャラが計画通り。0個/不足は不合格）
- [ ] 「器」だけ作って中身（演技・表情）が空になっていないか
- [ ] 前回までの**指摘・決定が反映**されているか（手戻り・蒸し返しがないか）
- [ ] 優先順位は正しいか（重要なものを先に作っているか）
- [ ] コスト・予算超過がないか／「完成」と言える状態か

### 評議員2 アートディレクター ─ 作画品質・統一感
- [ ] 基準キャラと**並べて同一人物に見えるか**（顔・髪型・体型）
- [ ] パワプロ準拠（2頭身／鼻・口・耳なし／指なし丸ミトン）
- [ ] **線**の太さ・色・質感が基準と一致
- [ ] **塗り**のセル影・ハイライトが基準と一致（髪・靴・手の艶含む）
- [ ] **配色**（肌・髪・スーツ・ネクタイ・シャツ・靴）が基準と**完全一致**＝生成ドリフトなし
- [ ] **後付け・貼り付けに見える箇所がない**（合成パーツが陰影なしで浮いていないか）
- [ ] 一枚の絵として破綻がない（手描きの完成度）

### 評議員3 キャラクターデザイナー ─ 演技・感情・個性（※今回の見落とし担当）
- [ ] **表情差分が実際に作られ、各々が一目で識別できる**（基本/笑い/怒り/困り/驚き 等）
- [ ] 各表情が**目・眉・頬だけで感情が伝わる**（笑い＝笑い目、怒り＝吊り眉…）
- [ ] ポーズと表情の**組み合わせが意味を成す**（喜びポーズ＋笑い顔 等）
- [ ] キャラの**個性**が出ている（主人公＝ツッコミ気質／矢田＝相方 の差）
- [ ] 感情のバリエーションが**必要十分**（足りない感情がないか）
- [ ] 並べて「キャラが生きているか／死んでいるか」の最終判断

### 評議員4 テクニカルアーティスト／QA ─ 実装・規約の機械チェック
- [ ] 透過：**連結成分=1**（孤立ノイズなし）・縁ハローなし（濃色背景に合成して確認）
- [ ] **頭の位置・大きさが基準に整列**（顔プレートがどのポーズにも重なる）
- [ ] レイヤー：体img＋顔imgが**ズレなく重なる**（プレビュー/実機で確認）
- [ ] 解像度・キャンバス規格、**命名規則**（§2）に従っているか
- [ ] §3 の A〜H 機械チェック（口・鼻・指なしは**拡大目視**／1体のみ）
- [ ] ファイルサイズ（モバイルで重すぎない）・キャッシュバスター（`?v=`）

### 評議員5 プレイヤー／UXレビュー ─ 実機・体感
- [ ] **スマホ実サイズ**で表情・ポーズが**判別できる**（潰れない）
- [ ] ゲームUI・背景の上で**浮かない**（馴染む）
- [ ] 差し替え時に**位置・サイズが飛ばない**（ガタつかない）
- [ ] 出来事と絵が一致して**気持ちよい**（ウケた→笑い顔 が腑に落ちる）
- [ ] 全体トーン（DESIGN.md／パワプロpop）と合う／「自分がプレイヤーなら嬉しいか」

### 運用ルール
1. 成果物ごとに5人が**独立に PASS/FAIL ＋ 具体理由**を出す。
2. **1人でも FAIL なら不採用** → 修正 → 再審。**全員 PASS で初めて採用**。
3. 報告は必ず **5人の評定表** で提示する。
4. 評議員は越権せず**自分の担当を厳しく**見る。
