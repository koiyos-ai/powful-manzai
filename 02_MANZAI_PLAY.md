## 4. 漫才プレイ(リズムゲーム) ✅ 実装済み

### 4.1 基本方式 ─ リズム天国スタイル
- 構成したネタを本番で披露する部分は**リズムゲーム**。
- **BGMのビート自体が譜面**。AudioContext の時計を唯一のマスタークロックとし、楽曲・ノーツ・判定・演出すべてを「拍(beat)」で統一管理 → 音ズレが原理的に起きない設計。
- ルックアヘッド・スケジューラ方式(25ms間隔・150ms先読み)。

### 4.2 操作と判定
- めくり札(パーツカード)が流れ、判定枠「**ココ!**」に重なる瞬間=**音楽のジャスト拍**でタップ。
- 画面のどこをタップしてもOK(スマホの片手操作前提)。
- **4段階判定**: ピタッ!(Perfect) / まずまず(Good) / ズレた(Bad) / …シーン(Miss)。
- 基準判定窓: **±[65, 120, 180]ms**(Perfect / Good / Bad、M札基準)。※2026-07-11シビア化(旧85/150/220→現行値。設計初期の95/185/290はさらに旧い)。値は仮、P5-01で最終調整。
- **サイズ別判定窓倍率(2026-07-11)**: S札(威力7-8=強パーツ)**×0.8**(ピタ±52ms) / M札×1.0 / L札(威力1-3)**×1.15**(ピタ±75ms)。強ネタほど判定がシビア=難易度で威力を買う設計。§4.3の「判定窓倍率」のサイズ軸を先行実装(タグ別倍率はP2-02)。※値は仮。
- 札の流れる速度: 判定ラインまで**3.5拍**(旧4拍。2026-07-11、札間隔の確保=重なり防止と速度アップを兼ねる)。
- **札の重なり防止(2026-07-11・見た目のみで判定タイミングは不変)**: ①長押し/連打札の尻尾は次の札の手前で止める ②裏拍札は少し上に浮かせて表示 ③通常札の幅は拍間隔を上限にする(狭い画面対策)。
- タップすべき拍にはブラス風スタブ和音、1拍前にウッドブロックの予告音 → 耳でも合わせられる。

### 4.4 会場温度システム ✅(2026-07-08 難易度設計を確定・v2実装済み)
- **初期値0**(客は冷えた状態から。ツカミで温める)。※旧仕様の30から変更(ユーザー決定)。フリープレイ(pm_honban無し)は常にこの0。
- **正規化単位 `u = 110 ÷ 総ノーツ数`**: 全ピタッ!の合計が110%相当(MAX100で頭打ち)になる傾斜。貯金は1割しかなく、シーンが出ると満点温度から脱落する。※2026-07-11ユーザー決定(旧: 100÷(総ノーツ×0.8)=125%相当は甘すぎた)。
- 変化量: ピタッ! **+1u** / まずまず **+0.5u** / ズレた **−1u**(減点) / シーン **−4u**(2026-07-11、-2.5uから強化)。ツカミ成功で**+6**(初回のみ)。※値は仮(P5-01で最終調整)。
- 目安(30枚構成・u≈3.7): シーン1回=−14.7+機会損失3.7=実質18超の減点 → 全ピタ貯金10では吸収不可。
- 温度が高いほど全パーツの笑い倍率が上がる(係数 `0.5 + temp/100`)。
- オチは温度で最終ボーナス(`×(1 + temp/200)`)。
- 客席(fal.ai生成の一枚絵)が温度で「着席静聴 ⇄ 着席のまま爆笑」にクロスフェード。拍でスウェイ、ウケで跳ねる。
- ✅ 温度バー(ウケメーター)は低温=青→黄→橙→高温=赤に色変化(旧改修指示③を消化)。
- ✅ 本番(劇場/M-1)は客入り(人気)で開始温度が変わる: `pm_honban.startTemp = round(S.pop × 0.25)`(pop0→0 / pop100→25)。フリープレイは0のまま。※値は仮(P5-01で最終調整)。P2-01完了。

### 4.5 笑い量の計算式 ✅
```
笑い量 = 威力 × 10
       × 判定倍率(Perfect 1.0 / Good 0.6 / Bad 0.25)
       × タグ笑い倍率
       × 基礎能力補正(対応能力 0〜100 → ×0.9〜×1.1)
       × 温度倍率(0.5 + temp/100)
       + 固有効果(ノリ引き継ぎ / シュール温度補正 / 相性ボーナス 等)
オチのみ最後に ×(1 + temp/200)
```

### 4.6 能力フックシステム(将来アップデート基盤) ✅(基盤実装済み)
- 主人公データ `PLAYER`(基礎能力5種 + 特殊能力リスト)を中心に置く。
  **育成パートはこのオブジェクトを書き換えるだけで本番に反映される**構造。
- **基礎能力補正**: 札の種別 → 対応能力を参照し、判定窓 ±20% / 笑い量 ±10% を常時補正。
- **特殊能力**: `onMiss` / `onPerfect` / `onGameStart` 等の**フック名で登録するデータ駆動方式**。
  条件を満たすと発動し、カットイン演出が出る。
  - 実装済みの例: **◯スベリ耐性**(スベった時の温度低下を半減)。
  - 設計済みフック例: ◎会場ジャック(連続Perfectで温度加算) / ◯つかみ(1番の判定窓+30%) / ✕テンパり(序盤2札の窓-20%)。
- 🔧 **未反映の改修指示**: 能力発動カットインは**画面左側**に出す(視認性が高いため)。

### 4.7 演出 ✅
- **キャラクター**: SVG描画のパワプロ風2頭身(巨大丸頭・点目・眉・ほっぺ・白手袋・白靴・ネクタイ付きスーツ)。ボケ=赤スーツ、ツッコミ=青スーツ。
- **アクション**: ツッコミ時はツッコミがボケを叩き(踏み込み+腕振り上げ)、ボケがのけぞって「ペシッ!」のコミック文字。ボケ時はジャンプ+両手上げ。ツカミ/フリ/オチは二人で弾む。
- **ビート連動**: 毎拍、判定枠が明滅・キャラが小刻みにボブ・観客が左右にスウェイ。画面全体がBPMで脈打つ。
- **カウントイン**: 専用カウントダウンを廃止し、イントロ4〜7拍目に「3・2・1・いくで!」をビートぴったりに表示。
- ✅ Perfectタイミングでウケた時、客の笑い声SE(WebAudio合成・温度が高いほど厚い)+客席から「ワハハ」文字が立ちのぼる(旧改修指示②を2026-07-08 v2で消化)。
- **リザルトの文言規約(2026-07-11確定)**: 漫才プレイのリザルト(ランクスタンプ・温度コメント)は**内心評価=演技の手応え**に限定する(「カンペキや!/ええ感じ!/ビミョー…/ガタガタや…」等)。「ウケた/スベった」等の**客の反応の断定は書かない**。ウケの確定評価(出来%・大ウケ/スべった・人気増減)は**育成側 `applyTheaterResult` が唯一の採点箇所**(acc×ネタ威力×実力×人気上限)。acc高×弱ネタ・低人気で育成側の評価が下がっても矛盾しないようにするため。本番時はリザルトに「※結果発表は戻ってから!」を添える。

### 4.9 音楽仕様 ✅
- BPM 128 / 4分の4拍子。Aマイナーペンタトニック。
- 構成: スクエア波リード + トライアングルベース + ドラム(キック/スネア/ハイハット)。すべてWebAudioで合成。
- 譜面構造: イントロ2小節(出囃子) → 8拍目からタップ開始 → (間)の休符地帯 → オチでクラッシュ+フィナーレ → アウトロ上昇フレーズ。総演奏時間 約13.6秒(プロト版)。
- 効果音: タップ音 / どつき音(ペチン) / 笑い声(ノイズバースト) / スベり音 / 歓声。
