## プロジェクト基本情報

- **作品名**: パワフル漫才(仮)
- **ジャンル**: パワプロ「サクセス」をお笑い芸人に置き換えた育成シミュレーション + リズムゲームのハイブリッド
- **対象プラットフォーム**: スマートフォン(iPhone/Safari 優先)、単一HTMLで動作
- **技術スタック**: HTML / CSS / Vanilla JS / WebAudio API(BGM・SE全合成)
- **現在の実装状況**: 漫才リズムゲーム部分が v6 までプロトタイプ実装済み。サクセスUIは試作v3まで(方向性確定)。
- **最終目標**: 全要素を統合し、アプリケーションとして本格プレイ可能にする

## 1. コンセプト / 世界観 ✅

### 1.1 基本コンセプト
パワプロのサクセスの骨格である「ターン制行動選択 → 能力育成 → 定期試験(試合) → 最終評価でキャラ完成 → エンディング」を、野球からお笑い芸人に置き換える。

### 1.2 主人公設定
- **芸歴7年目**の漫才師。吉本的な大手事務所に所属。
- 「**10年目までにM-1風賞レースで優勝できなければ引退**」という約束を背負っている。
- そのため、プレイ期間は **3年間**(7年目→10年目)。
- **主人公の名前はプレイヤーが入力する**。
- 出身は大阪(関西弁)。→ 詳細は `04_STORY.md`。

### 1.3 ゴール
- **M-1グランプリ風の賞レースで優勝すること**が最大の目標。
- ただし優勝以外にも多彩なエンディングが用意され、エンド回収が周回動機となる(→ `04_STORY.md` のエンディング図鑑)。

## 8. 周回プレイ設計 ✅(方針)

周回動機の3軸:
1. **コレクション**: エンディング図鑑、コンボレシピ図鑑のコンプリート。
2. **別ルート**: 相方・ライバルとの関係分岐、芸風(系統)特化ビルドの作り分け。
3. **やり込み**: 縛りプレイ・高難度挑戦(💡 将来)。

- エンディング図鑑の詳細 → `04_STORY.md`
- コンボレシピ → `03_NETA.md`

## 付録B. 確定した重要な設計判断の記録
- パーツはテキスト化せず抽象表現(実漫才に面白さで敵わない & 会話成立が困難)。→ `03_NETA.md`
- 威力は【1〜9】(基礎能力 G〜S と視覚的に区別)。
- 性質タグは各種別5つまで(構成難易度の上限管理)。
- ネタ合わせでパーツ強化はしない(ハクスラ的緊張感の維持)。
- 評価ゲージは人気のみに一本化。
- 1年目は相方マイナスイベント免除(学習曲線)。
- 敗者復活は演出のみ・プレイヤー利用不可。
- 決勝は別ネタ2本必須。
- エンド判定は突出能力重視・閾値80以上・上から評価方式。
- 準決勝進出+総合一定以上は引退回避(バラエティ成功路線)。
- 約束の相手は恋人(大阪の幼なじみ)。約束時期はプレイヤーが7年目開幕で決意する折衷案。→ `04_STORY.md`
- 彼女システムは不採用(実装難易度の抑制)。

### 育成・ネタガチャ・発想力サイクル(再設計・確定)→ 詳細 `research/neta-gacha-design.md`
- **育成の中心思想**: 「芸人は舞台・稽古・生活で育つ」。抽象トレーニングを撤廃し、発想力＝経験(ひらめき)を稽古/インプット/遊び/本番から稼ぎ、ネタ(収集)と腕(能力)に分配する。
- **ネタ作り＝ガチャ**: 発想力は**固定コスト**。**レア度(威力分布)は基礎能力の合計値が駆動**(`a=(合計-100)/400`、cMin1.5/cMax7.6/σ1.6)。MAXで威力9≈19%・序盤はほぼ0%。※`03_NETA §3.2` を上書き改訂予定。
- **グレードはS/A/B/C/D/E/F/Gの8段階**(G<20 / F20 / E40 / D50 / C60 / B70 / A80 / S90)。※ゲーム本体の`gradeOf`はE止まり→F/G追加で拡張。
- **能力成長＝グレード段階の逓増コスト**(+1あたり G5/F6/E8/D10/C13/B17/A22/S30 発想力)。
- **初期能力＝100点を5能力に乱数分配**(個別上限40・E1つで当たり)。成長レンジ100〜500。最終到達目安: 平均A2B2C1≈370 / 当たりS2A2≈425。
- **ネタ作り・腕を磨くは週を消費しない(即時)**。週を消費するのは稼ぐ/回復/本番のみ。決勝の別ネタ2本要件と両立させるため。
- **劇場出番の発想力報酬＝内部得点比例**(`base + k×内部得点`)。周回者は入手発想力から決勝点を読める(知識資産)。
- **ゴミパーツ対策＝ミキサーガチャ(実装)** / パーツ分解→発想力(将来案・lossy)。
- 数値調整は `research/gacha-tuner.html` で検証する。
